Detailed Description

A simple wrapper around a boolean, mutex, and condition variable for concise thread synchronization. Each call to set() will wake at most one thread. If no threads are waiting when set() is called, nothing will happen and the next to wait will still block.

Note
In the case of multiple waiters, it is possible for set() to wake a thread that began waiting after the call to set().
This is because set() releases the mutex before calling condition_variable::notify_one, so it is possible (but unlikely) that a waiter will acquire the mutex and check m_set_count within this period.
