Detailed Description

Functionally similar to jack::unicast_event except for two differences. Firstly, set() calls wake all currently waiting threads. Secondly, between a call to set() and the last waking thread releasing the mutex, new waiters are temporarily blocked from the waiting pool. This is to prevent already-woken waiters from returning to the waiting pool and spuriously waking again.
